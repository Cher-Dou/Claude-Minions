# Devlog: 2026-07-21

## What changed

- Added the first `lit-scout` evaluation scaffold under `evals/`.
- Added a shared `evals/metrics.py` module so future agent benchmarks can reuse
  the same scoring logic.
- Added `research-manager` as the top-level orchestration agent.
- Added `docs/agent-architecture.md` to explain routing, quality gates, and the
  roadmap toward memory, evidence scoring, and a knowledge graph.
- Added `benchmarks/AIToxBench/` as the long-term benchmark roadmap.

## Why it changed

The project is moving from "useful toxicology agents" toward a measurable AI
research platform. Evaluation needs to sit beside the agents as a first-class
system, not as an afterthought.

## What this enables

- Reproducible claims about agent performance.
- Easier regression checks when agents or prompts change.
- A stronger portfolio story: the project measures scientific reliability, not
  only automation.
